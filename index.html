Highlight #3 README File

In Lesson 3 I learned about what a “ReadMe” file is and why it is important. A “ReadMe” file is a short file included with some software written by the software’s author. It contains basic information about the software and about the who the author is. Things like the date, author’s name, and a brief summary of the software are included so people can have some background information on how the code was written and what it is intended to to. This is interesting because it is like a hidden about the author that coders know about. This is relevant to our class because it is another step in knowing the basics of what writing code includes.

Highlight #4 Git Commands

In Lesson 3 we learned new commands to type into the terminal of cloud 9. New commands that were taught include “git init”,”git add .”, and “git commit -m (descriptive message)”. The first command, “git init” is used to create a new repository in github. When “git add .” is entered in the terminal it brings all of the files in the section to the git to be used later. Lastly “git commit –m” is used to make a snapshot and basically tells when something should be read as a message. This is interesting to me because it is new but makes sense to me about why these commands are used. These are important because they are crucial commands that need to be used in order to make things work.

Highlight #5 Unzipping Files

Lesson 3 made it clear to me why it is important to upload the unzipped version of a file to cloud 9 when creating a new workspace. When taking an online file that is compressed into a zipped file it must be downloaded and then extracted, turning it back into all of the individual files that make up the whole file. This is important because it makes it so each file can be edited individually. Files can be zipped or compressed when there are many files that need to be sent to the same place, it makes it easier to move everything and takes less storage. Zipped files interest me because they seem like an efficient way to organize data.
